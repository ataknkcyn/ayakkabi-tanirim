## Codebase Patterns
- Pure HTML/CSS/Vanilla JS project — no build tool, no package.json, open index.html directly in browser
- Tests use Node.js ESM (import/export), run with: node tests/<test-file>.js
- Git commits: `git -c user.email="developer@openclaw" -c user.name="Developer" commit -m "..."`
- No .env or secrets — API key stored in browser localStorage only
- Test pattern: mock browser APIs (localStorage, DOM) via pure JS objects in Node; also do source-text assertions on HTML/JS files
- CSS classes for new UI rows: use BEM-ish flat naming (.api-key-row, .btn-icon, .btn-save, .api-key-saved)
- `[hidden]` CSS utility is in styles.css — use `element.hidden = true/false` in JS, never touch display directly
- showSavedMessage() uses a timer stored on the function itself (showSavedMessage._timer) to cancel previous timers — useful pattern for debounced UI feedback
