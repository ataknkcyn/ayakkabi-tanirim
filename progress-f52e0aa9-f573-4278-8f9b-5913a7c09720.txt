## Codebase Patterns
- Pure HTML/CSS/Vanilla JS project — no build tool, no package.json, open index.html directly in browser
- Tests use Node.js ESM (import/export), run with: node tests/<test-name>.js
- Git commits: `git -c user.email="developer@openclaw" -c user.name="Developer" commit -m "..."`
- No .env or secrets — API key stored in browser localStorage only
- Module-level state variables (selectedFile) declared outside DOMContentLoaded, handlers inside
- DOMContentLoaded wraps all DOM manipulation; module-level variables declared above it
- Test files parse source files as strings (readFileSync + regex) — no DOM/browser runtime needed
- CSS `.drag-over` class applied via JS; styled in CSS with `#drop-zone:hover, #drop-zone.drag-over { ... }`
- Hidden `<input type="file">` + `fileInput.click()` pattern for programmatic file picker trigger
- FileReader + readAsDataURL for image preview before upload
- async/await with FileReader: wrap readAsDataURL in a Promise for clean async usage in async functions
- Button enable/disable state: use a central `updateAnalyzBtn()` helper called from every state-changing point (file select, key input, file clear)
- Test regex for multi-line code: replace `/\n/g, ' '` before testing cross-line patterns, or use `[\s\S]` for multiline
- Anthropic browser access: requires `anthropic-dangerous-direct-browser-access: true` header for direct browser fetch calls
