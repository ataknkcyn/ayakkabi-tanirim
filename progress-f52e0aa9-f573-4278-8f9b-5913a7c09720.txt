## Codebase Patterns
- Pure HTML/CSS/Vanilla JS project — no build tool, no package.json, open index.html directly in browser
- Tests use Node.js ESM (import/export), run with: node tests/<test-name>.js
- Git commits: `git -c user.email="developer@openclaw" -c user.name="Developer" commit -m "..."`
- No .env or secrets — API key stored in browser localStorage only
- Module-level state variables (selectedFile) declared outside DOMContentLoaded, handlers inside
- DOMContentLoaded wraps all DOM manipulation; module-level variables declared above it
- Test files parse source files as strings (readFileSync + regex) — no DOM/browser runtime needed
- CSS `.drag-over` class applied via JS; styled in CSS with `#drop-zone:hover, #drop-zone.drag-over { ... }`
- Hidden `<input type="file">` + `fileInput.click()` pattern for programmatic file picker trigger
- FileReader + readAsDataURL for image preview before upload
- Loading state functions (showLoading/hideLoading) defined at module level (before DOMContentLoaded) so they can reference DOM via getElementById at call time — they do NOT cache DOM refs at module scope
- Use `style="display:none"` (inline style) for initially-hidden sections that JS will toggle via .style.display = 'flex' — avoids [hidden] attribute conflict since [hidden] uses display:none !important
- try/finally pattern for API calls: showLoading() before fetch, hideLoading() in finally block
- Module-level helper functions (displayResults, showLoading, hideLoading, showError, hideError) use getElementById at call time, NOT cached at module scope
- Result card uses `.result-card` class (not ID) so it can be reused; section uses `#result-section` ID for JS targeting
- `style.display` toggling (not `hidden` attribute) for sections controlled by JS — consistent approach throughout
- Error handling pattern: throw custom errors with `err.status = response.status` and `err.isJsonError = true` flags so catch blocks can map to Turkish messages without string matching
- app.js is a plain (non-module) script → module-level functions are on `window`, so HTML onclick="hideError()" works
- Turkish error messages defined as module-level constants (ERR_NO_FILE, ERR_NO_API_KEY, etc.) for testability
- Dismiss buttons in error sections use `onclick="hideError()"` attribute (works since app.js is non-module global script)
